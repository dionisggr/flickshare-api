FlickShare API

Congrats!

You've successfully arrived at the FlickShare API!


Resources

  • /users x5
  • /projects x10
  • /issues x5

To access and modify resources, you can use any HTTP method:

  • GET POST PATCH DELETE

Endpoints that require Authentication

Closed endpoints that require a valid username and password to be included in the header body of the request.

  • Login: POST /login
    • 'Admin' credentials (or you may Register)
      • Username: admin
      • Password: password

Endpoints that require Authorization

Closed endpoints that require a valid JSON Web Token or API key to be inlcuded in the header 'Authorization' of the request.

For 'Admin' privileges:

// Add to request header headers: {'Authorization': 'Bearer <JSON Web Token>'} // For Demo purposes, you may use: 'Bearer my-secret-key' // For Admin Demo purposes, you may use: 'Bearer my-secret-admin'

If sending content through request body (POST, PATCH), don't forget to add the following in the headers:

// Add to request header headers: {'Content-Type': 'application/json'}

Open Endpoints

Endpoints that require no Authentication.

  • Create User (Register): POST /api/users
  • Login: POST /api/login
  • Get Main Lists: GET /api/lists/main
  • Get All Movies: GET /api/movies

Closed Endpoints

Endpoints that will require prior Authentication (user login) for access.

User related

Each endpoint manipulates information related to users.

  • Get User Info: GET /api/users/:user
  • Edit User Info: PATCH /api/users/:user
  • Delete User Account: DELETE /api/users/:user

List related

Each endpoint manipulates information of general and user lists.

  • Create a List: POST /api/lists
  • Get List: GET /api/lists/:list
  • Edit List: PATCH /api/lists/:list
  • Delete List: DELETE /api/lists/:list

Movie related

Each endpoint manipulates information related to movie data.

  • Add Movie to Database: POST /api/movies
  • Get Movie: GET /api/movies/:movie

Movie-List related

Each endpoint manipulates information related to the movies in lists.

  • Get List Movies: GET /api/movies/lists/:list
  • Add Movie to List: POST /api/movies/lists/:list
  • Delete Movie from List: DELETE /api/movies/:movie/lists/:list

Access / Token related

Each endpoint manipulates information related access / token management.

  • Login: PATCH /api/login
    • Refresh JWT Token: PATCH /api/token

    Admin related

    Each endpoint manipulates information related to all data, only able to be accessed by an Admin user.

    Admins can manipulate all previous endpoints as well.

    • Get All Users: GET /api/users
    • Get All Lists: GET /api/lists
